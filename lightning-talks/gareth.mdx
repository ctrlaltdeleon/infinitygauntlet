// ## Why Speak?

// ---

// ### "Abstract" Reasons

// <Appear>
// <p>Reinforce and demonstrate your knowledge</p>
// <p>Community participation - involve yourself and give back</p>
// <p>Challenge yourself - leave your comfort zone</p>
// <p>Show off cool stuff you've done</p>
// </Appear>

// ---

// ### "Practical" Reasons

// <Appear>
// <p>Communicating technical concepts is hard - great practice</p>
// <p>
// Excellent resume and portfolio material - slides, turn it into a blog post
// </p>
// <p>Stand out in interviews</p>
// <p>Free networking/chance to show you know your stuff</p>
// </Appear>

// ---

// ## How & Where to Speak

// ---

// ### Meetups

// <Appear>
// <p>Contact an organizer via Slack or at an event</p>
// <p>SDJS organizers are great and will be enthused you want to speak!</p>
// <p>Find the right Meetup for your topic - SDJS Monthly is "general"</p>
// <p>5 minute lightning talks are great starting place</p>
// <p>We are lucky to have this awesome community!</p>
// </Appear>

// ---

// ### Conferences

// <Appear>
// <p>Search for conferences in your area and/or relevant to your topic</p>
// <p>Submit to the Call for Proposals, be prepared for rejection</p>
// <p>Sometimes speaker travel, lodging, meal fees are covered</p>
// <p>Helps if you are a known quantity - start at Meetups!</p>
// </Appear>

// ---

// ## What to Speak About

// ---

// ### "I don't have anything to speak about"

// <Appear>
// <p>Wrong!</p>
// <p>You have something to speak about!</p>
// <p>React is a huge topic, JS is much bigger, web dev is gargantuan</p>
// </Appear>

// ---

// ### So what do I speak about?

// <Appear>
// <p>Anything you work with or have used or think is cool</p>
// <p>
// OK if it "seems obvious" to you - chances are it won't be for someone else
// </p>
// <p>Audience has different skill levels; cater to any of them</p>
// <p>Pick something - anything - and make 5 min worth of slides about it</p>
// </Appear>

// ---

// ### Such as...

// <Appear>
// <p>"5 things I wish I knew before learning React"</p>
// <p>"Why I chose React for project X"</p>
// <p>"Moving from jQuery to React"</p>
// <p>"MobX for Redux Users"</p>
// <p>"Animating Components in React"</p>
// <p>"Asynchronous Data Fetching in React"</p>
// <p>
// Literally - seriously -{" "}
// <strong>anything relevant to the event topic</strong>
// </p>
// </Appear>

// ---

// ### Don't be Negative

// <Appear>
// <p>Bad: "Why You Should Switch From React to Vue"</p>
// <p>Good: "Understanding Vue for React Developers"</p>
// <p>Bad: "Lol You Still Use jQuery in 2019?"</p>
// <p>Good "Why You Might Not Need jQuery in 2019"</p>
// </Appear>
